How the prompts auto-generate
Template approach — zero manual effort:
Given a focused node with title, type, and parent title:

Prompt 1: "Tell me about {title}" — always works, dead simple
Prompt 2: "How does {title} relate to {parent}?" — uses graph context
Prompt 3: "What does this say about Sebastiaan?" — personal angle (only for relevant types)

The chat itself handles translations — the LLM responds in whichever language the user has set. For the prompt labels, you can use simple i18n templates like t("ui.tell_me_about", {title}) with just 2-3 translation keys (vs. hundreds of per-node prompts).

Pillar colors are mapped by walking the containment-edge tree upward from any node to its root pillar (career/education/hobbies/community/personality/nb-work). All children inherit the parent's pillar color. No data model changes needed.
