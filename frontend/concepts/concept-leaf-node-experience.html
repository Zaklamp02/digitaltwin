Leaf Node Experience — Concept Mockups

When a user reaches a leaf node (no children to expand), the graph currently feels like a dead end. Below are 4 concepts for making that moment feel rich and inviting — showing content teasers, linked documents/images, and conversation starters. All concepts work with the existing data model (node.body, node.metadata, edges, image_path).

A — Slide-up Detail Card
A card slides up from the bottom when a leaf node is focused. Shows a content teaser (first ~200 chars of node.body), image thumbnails, document chips, and conversation starter prompts.
Clicking a prompt starts a chat about that topic.

B — Contextual Bubble Prompts
The lightest-weight option. When a leaf node is focused, 2-3 chat-bubble-style conversation starters animate in near the node. Clicking one immediately starts a chat about that topic. No content preview — the conversation IS the content discovery.
Optionally: a small "📄 has content" indicator on the node itself.

C — Node Bloom (expanded node + satellite media)
The focused leaf node "blooms" — expands into a larger circle showing a content excerpt. Around it, satellite nodes orbit: pink for images (showing actual thumbnails), gray for documents, and indigo for related cross-linked nodes. A chat button sits below.

D — Context Pane (graph + side panel) recommended
When a leaf node is focused, a side panel slides in from the right, showing the full node detail: type badge, content preview, image grid, document list, and conversation starters. The graph stays visible and interactive on the left. This is the most complete solution — it shows everything without obscuring the graph, and naturally guides users into conversation. On mobile, it becomes a bottom sheet (like Concept A).

Strengths
  • Most complete — content, images, docs, prompts all visible
  • Graph stays visible and interactive (no context loss)
  • Panel can scroll for long content independently
  • Natural flow: explore graph → see content → start chat
  • Works well with both content-rich and content-light nodes

Considerations

  • Graph needs to re-center when panel opens (camera shift)
  • Requires a new API call to fetch node.body (currently only /api/graph returns limited fields)
Implementation Notes
Data already available: The existing /api/graph endpoint returns has_document per node. For content preview, we'd add a lightweight endpoint like GET /api/node/{id}/preview returning: first 300 chars of body, list of image_paths, list of document filenames, and edges. No data model changes needed.

Conversation starters can be generated from: node title + parent title + node type. Templates like "Tell me about {title}", "How does {title} connect to {parent}?", "What's interesting about {title}?" — or optionally generated by the LLM for richer prompts.

Hybrid approach possible: Concepts can be combined. For example, use D (Split Pane) on desktop and A (Detail Card) on mobile. Or use B (Bubble Prompts) for nodes without content and D for nodes with rich content.
